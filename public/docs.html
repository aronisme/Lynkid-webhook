Dokumentasi Integrasi Webhook

Panduan bagi developer untuk menerima dan memproses data dari Webhook Hub.

1. Pengantar

Webhook Hub bertindak sebagai penerus (router). Ketika transaksi berhasil terjadi di Lynk.id, Webhook Hub menerima data tersebut dan meneruskannya (HTTP POST) ke URL Proyek Anda berdasarkan Routing Rules yang telah dikonfigurasi.

Penting: Payload yang dikirimkan oleh Webhook Hub adalah persis sama (passthrough) dengan payload asli dari Lynk.id. Signature validation telah dilakukan oleh Hub, sehingga server tujuan Anda tidak perlu melakukan validasi signature lagi.

2. Request Format

Webhook Hub akan mengirimkan HTTP POST request ke endpoint (URL Tujuan) Anda.

HTTP Headers

  • Content-Type string
    Selalu application/json
  • User-Agent string
    Selalu LynkidWebhookHub/1.0
  • X-Webhook-Source string
    Selalu lynkid-hub. Bisa digunakan untuk memverifikasi sumber request.
  • X-Webhook-Resend boolean
    Bernilai true jika request ini adalah hasil klik tombol "Resend" secara manual di Dashboard. Jika webhook asli, header ini tidak ada.
  • (Custom Headers)
    Header tambahan apapun yang Anda definisikan di pengaturan Rule (misal: Authorization: Bearer token123).

Body (JSON Payload)

Struktur di bawah ini adalah format standar dari Lynk.id.

{
  "event": "payment.received",
  "data": {
    "message_action": "SUCCESS",
    "message_code": "0",
    "message_id": "API_CALL_...",
    "message_data": {
      "refId": "412039de85...",
      "createdAt": "2026-03-22T05:09:37",
      "customer": {
        "email": "pembeli@email.com",
        "name": "Nama Pembeli",
        "phone": "08123456789"
      },
      "items": [
        {
          "uuid": "69bf16671a403958...",
          "title": "Nama Produk",
          "price": 100000,
          "qty": 1
        }
      ],
      "totals": {
        "grandTotal": 100000,
        "customerPay": 100000
      }
    }
  }
}

Field Penting yang Biasa Digunakan:

  • data.message_data.customer.email: Email pembeli untuk aktivasi akun/layanan.
  • data.message_data.items[0].uuid: ID Produk unik dari Lynk.id untuk verifikasi produk apa yang dibeli.
  • data.message_data.refId: Referensi transaksi unik.

3. Contoh Implementasi Penerima (Receiver)

Berikut adalah contoh kode menggunakan Node.js (Express) untuk menerima webhook dari Hub.

const express = require('express');
const app = express();

// Parse JSON body
app.use(express.json());

// Endpoint yang didaftarkan di Webhook Hub (URL Tujuan)
app.post('/api/webhook/lynkid', async (req, res) => {
    // 1. Verifikasi sumber request (Opsional, tapi direkomendasikan)
    if (req.headers['x-webhook-source'] !== 'lynkid-hub') {
        return res.status(403).json({ error: 'Unauthorized source' });
    }

    // (Opsional) Verifikasi custom token jika Anda setel di Rule Headers
    // if (req.headers['authorization'] !== 'Bearer secret_token') ...

    const payload = req.body;

    // 2. Pastikan ini adalah event sukses
    if (payload.event !== 'payment.received' || payload.data.message_action !== 'SUCCESS') {
        return res.status(200).send('Ignored non-success event');
    }

    try {
        const customerEmail = payload.data.message_data.customer.email;
        const items = payload.data.message_data.items;
        const refId = payload.data.message_data.refId;
        
        console.log(`[Webhook] Menerima pembayaran dari ${customerEmail} (Ref: ${refId})`);

        // 3. Proses aktivasi produk
        for (const item of items) {
            console.log(`Mengaktifkan produk: ${item.title} (UUID: ${item.uuid})`);
            
            // TODO: Logika database/aplikasi Anda disini
            // await activateUserPremium(customerEmail, item.uuid);
        }

        // 4. Selalu respon dengan 200 OK agar Hub tahu pengiriman berhasil
        return res.status(200).json({ success: true, message: 'Webhook processed' });

    } catch (error) {
        console.error('Webhook error:', error);
        // Respon dengan 500 akan membuat Hub mencatat status FAILED
        return res.status(500).json({ error: 'Internal server error' });
    }
});

app.listen(3000, () => console.log('Receiver berjalan di port 3000'));

4. Tanya Jawab (FAQ)

Apa yang terjadi jika server saya down (merespon 500 atau timeout)?

Hub akan mencatat status pengiriman sebagai FAILED atau PARTIAL_FAIL. Webhook tidak akan dicoba ulang secara otomatis, namun admin dapat masuk ke Dashboard Hub dan menekan tombol Resend secara manual kapan saja.

Berapa lama batas waktu (timeout) request?

Hub menunggu maksimal 15 detik. Jika server tujuan Anda butuh waktu proses lebih lama dari itu, sangat disarankan agar server Anda membalas 200 OK terlebih dahulu secepatnya, dan memproses data di background.
