Privacy Policy

Scraper Prompt Generator

Last Updated: April 8, 2026

This Privacy Policy explains how the "Scraper Prompt Generator" Chrome extension (the "Extension") collects, handles, stores, and shares user data. This policy applies to the live extension, the source package, and the build distributed for Chrome Web Store review.

The Extension is designed to minimize data collection and only access data needed to deliver user-requested features.

1. Data We Collect

1.1 Data You Provide

  • Google sign-in information: If you choose to sign in, the Extension uses chrome.identity and Firebase Authentication to process your email address, display name, Google account identifier, and authentication tokens.
  • API keys you enter: If you add your own Gemini API key or Groq API key, the Extension stores those keys in Chrome extension storage so it can send requests to the provider you selected.

1.2 Data Collected When You Use the Core Features

  • Page content you intentionally scrape: When you click "Scrape Images" or use the Extension's context menu, the Extension reads image URLs, image dimensions, image alt text when available, and limited page metadata such as the page title from the active tab.
  • Images and prompt inputs you choose to process: If you upload an image or select scraped images for analysis, the selected image data and your prompt settings are processed so the Extension can generate prompts.
  • Usage counter data: The Extension records account-related usage totals, such as image processing count, to enforce free-tier limits and determine premium access.
  • Shutterstock Trends page integration: If you open the supported Shutterstock Trends page, the Extension may inject interface buttons on that page so you can send a keyword into the Extension workflow. This behavior is limited to the supported Trends page pattern.

1.3 Data We Do Not Collect for Our Own Servers

  • We do not collect or store your browsing history for advertising purposes.
  • We do not sell personal information or scraped website content.
  • We do not store your private Gemini or Groq API keys on our own backend servers.

2. How We Handle and Use Data

  • Authentication: Sign-in data is used only to authenticate you, maintain your session, and associate your account with subscription and usage information.
  • Prompt generation: Selected images, image URLs, and related prompt inputs are used only to generate AI prompts at your request.
  • Feature delivery: Page content is accessed only after you trigger the feature on the current page or active tab.
  • Account management: Email, name, usage count, and premium status are used to manage freemium and premium functionality.
  • Settings persistence: API keys, language choice, provider choice, and similar settings are stored so the Extension can restore your preferences.

3. Where Data Is Stored

  • Chrome extension storage: API keys and most user preferences are stored in chrome.storage.sync. Cached login flags and Firebase session tokens are stored in chrome.storage.local.
  • Firebase / Firestore: Account records such as email address, display name, user ID, premium status, usage count, and account creation timestamp are stored on Google Firebase infrastructure.
  • Temporary browser memory: Scraped image results, uploaded files, and generated prompt results may be held temporarily in the browser while the Extension is open and processing your request.

4. Data Sharing and Disclosure

We do not sell, rent, or trade your personal data.

Data is shared only when required to provide the feature you asked for:

  • Google Gemini API: If you choose Gemini, selected image data and prompt content are sent directly to Google's AI API to generate results.
  • Groq API: If you choose Groq, selected image data and prompt content are sent directly to Groq's API to generate results.
  • Firebase services: Authentication data and account-related records are processed by Google Firebase and Firestore to manage login state, usage, and premium status.

Scraped page content is not shared with unrelated third parties, and it is only transmitted to the AI provider you actively selected when you start a generation request.

5. Data Retention

  • Local settings and API keys: Remain in Chrome storage until you remove them, clear extension storage, or uninstall the Extension.
  • Firebase account records: Remain in Firebase until you request deletion or we remove them for operational reasons.
  • Temporary processing data: Scraped page data, uploaded files, and generated results are kept only as long as needed for the current session unless you save them yourself outside the Extension.

6. Your Choices and Controls

  • Do not sign in: You can choose not to use Google sign-in features.
  • Remove API keys: You can delete saved API keys from the Extension settings.
  • Clear local data: You can clear Chrome extension storage or uninstall the Extension to remove locally stored settings and tokens.
  • Request account deletion: You can request deletion of your Firebase account record and related account data by contacting us.

7. Chrome Permissions Used

  • activeTab: Used to access the current page only when you trigger scraping or analysis.
  • <all_urls>: Used so the Extension can work on pages you choose to analyze.
  • scripting: Used to inject the extraction logic into the current page at your request.
  • storage: Used to store settings, API keys, login flags, and local session data.
  • identity: Used to support secure Google sign-in.
  • contextMenus: Used to add the right-click action for starting scraping.
  • sidePanel: Used to show the Extension interface in the Chrome side panel.
